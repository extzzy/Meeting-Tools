</p>

[English version](README.md)

Meeting Tools — плагин Obsidian для быстрого и единообразного создания Minutes of Meeting (MoM), то есть протоколов и итогов встреч. Он добавляет формы на основе Markdown-шаблонов, упрощённый журнал встреч и дашборд задач.

## Зачем нужен Meeting Tools?

Расширение создавалось для быстрого составления полезного MoM по встречам. Вместо ручного создания файла, выбора папки, копирования заголовков, форматирования задач и переименования заметки вы заполняете компактную форму, а Meeting Tools создаёт Markdown-файл по заданным правилам.

Если требуется зафиксировать только факт встречи, дату и краткий итог, используется **Быстрая встреча**. Для полноценного MoM с участниками, решениями, задачами и произвольными разделами используется **Новая встреча**.

## Возможности

- создание структурированных протоколов из Markdown-шаблонов;
- встроенный минимальный шаблон, если внешних шаблонов нет;
- структура файлов по году и месяцу, только по году или без подпапок;
- динамическая форма задач с темой, исполнителем и сроком;
- быстрый журнал записей формата «дата + итог»;
- общий дашборд задач с поиском, фильтрами и сменой статуса;
- добавление самостоятельных задач непосредственно из дашборда;
- независимая настройка папок шаблонов, протоколов, журналов и задач;
- русский и английский интерфейс с автоматическим выбором языка Obsidian;
- локальная работа через Obsidian Vault API без сервера, контейнера, аккаунта и сетевых запросов.

## Что такое Vault в Obsidian?

**Vault**, или хранилище Obsidian, — это обычная директория на вашем компьютере, которую вы открываете в Obsidian. В ней лежат Markdown-файлы (`.md`) с заметками, вложения и скрытая папка `.obsidian` с настройками и плагинами.

Например:

```text
Рабочие заметки/                    ← папка Vault
├── .obsidian/                      ← настройки и плагины Obsidian
├── Шаблоны/                        ← необязательные шаблоны Meeting Tools
├── Встречи/                        ← заметки, создаваемые Meeting Tools
└── Проекты/
```

Если в инструкции предлагается скопировать файл в Vault, его нужно поместить внутрь этой директории.

## Установка

### Через официальный каталог

После публикации Meeting Tools в официальном каталоге:

1. Откройте **Настройки → Community plugins** в Obsidian.
2. Нажмите **Browse** и найдите **Meeting Tools**.
3. Нажмите **Install**, затем **Enable**.

### Ручная установка

1. Скачайте `main.js`, `manifest.json` и `styles.css` из одной версии GitHub Release.
2. Создайте внутри Vault папку:

   ```text
   .obsidian/plugins/meeting-tools/
   ```

3. Поместите в неё три скачанных файла.
4. Перезапустите Obsidian.
5. Включите **Meeting Tools** в разделе **Настройки → Community plugins**.

Не переносите `data.json` между Vault: этот файл содержит настройки конкретного хранилища.

## Быстрый старт

После включения плагина в левой боковой панели появляются три кнопки:

- **Новая встреча** создаёт полноценный MoM;
- **Быстрая встреча** добавляет дату и итог в журнал;
- **Дашборд задач** показывает задачи из настроенной папки.

Папки и структуру файлов можно выбрать в **Настройки → Community plugins → Meeting Tools**.

Пустая папка полноценных протоколов или быстрых журналов означает корень Vault. Например, при пустой папке протоколов и структуре «Только год» файл будет создан в `2026/<встреча>.md`, а быстрый годовой журнал — непосредственно в `2026.md`.

## Шаблоны

Meeting Tools читает Markdown-файлы, расположенные непосредственно в настроенной папке шаблонов. Скопируйте нужные файлы из [`examples/templates/ru`](examples/templates/ru) или [`examples/templates/en`](examples/templates/en) в любую папку внутри своего Vault, затем выберите её в настройках плагина.

Пример:

```text
Мой Vault/
└── Шаблоны/
    ├── Встреча.md
    ├── Daily.md
    └── Еженедельный обзор.md
```

YAML-блок определяет метаданные. Markdown-заголовки становятся полями формы, а HTML-комментарий под заголовком — подсказкой поля:

```markdown
---
tags:
  - area/work
  - type/meeting
project: "Команда"
series: "Еженедельный обзор"
cadence: weekly
---

# Еженедельный обзор

> [!summary] Итог недели
> <!-- Главный результат и текущий риск. -->

## Решения

<!-- Зафиксируйте принятые решения. -->

## Задачи

<!-- Этот раздел станет динамической формой задач. -->

## Заметки

<!-- Дополнительный контекст. -->
```

Разделы `## Задачи` и `## Tasks` распознаются как динамическая форма задач. Файлы `Встреча.md` и `Meeting.md` считаются универсальным типом: форма дополнительно предлагает указать название встречи.

Пользовательские шаблоны не переводятся автоматически: заголовки и подсказки остаются на языке исходного файла. Встроенный шаблон следует выбранному языку интерфейса и остаётся доступным для ручного выбора даже при наличии внешних шаблонов.

## Журнал быстрых встреч

Быстрая встреча сохраняет только дату и введённый итог. Однострочная запись выглядит так:

<p align="center">
  <img src="assets/screenshots/quick-meeting.png" alt="Форма быстрой встречи" width="720">
</p>

```markdown
14.07.2026 — обсудили архитектуру и согласовали следующий этап
```

Доступны три структуры журнала:

- год/месяц: `<папка>/2026/07.md`;
- только год: `<папка>/2026.md`;
- без подпапок: `<папка>/Встречи.md`.

## Дашборд задач

<p align="center">
  <img src="assets/screenshots/task-dashboard.png" alt="Дашборд задач Meeting Tools" width="1200">
</p>

Созданные через форму задачи являются обычными Markdown-строками с чекбоксами:

```markdown
- [ ] Подготовить план внедрения — @Иван #исполнитель/иван 📅 2026-07-20
- [x] Проверить архитектуру — @Мария #исполнитель/мария ✅ 2026-07-14
```

Дашборд показывает задачу, исполнителя, срок, исходный файл и статус. Доступны поиск, фильтры по статусу и исполнителю, статистика просроченных задач и изменение статуса. Перед записью плагин проверяет, что исходная строка задачи не была изменена.

Дашборд можно использовать независимо как локальный task tracker. Укажите в настройках любую папку Vault с поддерживаемыми Markdown-строками задач: файлы не обязаны быть протоколами встреч или создаваться через Meeting Tools.

Кнопка **Добавить задачу** создаёт задачу с темой, исполнителем и сроком. Целевой Markdown-файл настраивается отдельно в разделе **Настройки → Meeting Tools → Дашборд задач** и должен находиться внутри папки поиска дашборда.

## Конфиденциальность и доступ к данным

Meeting Tools:

- не использует телеметрию и аналитику;
- не требует аккаунта или оплаты;
- не выполняет сетевые запросы;
- не обращается к файлам за пределами текущего Vault;
- читает шаблоны и задачи только из папок, указанных в настройках;
- записывает данные только при создании встречи, добавлении записи в журнал или явной смене статуса задачи.

## Разработка и выпуск

Исходники находятся в папке [`src`](src). Для разработки нужны Node.js 18+ и pnpm. Один раз установите зависимости:

```bash
pnpm install
```

Соберите минифицированный production-bundle:

```bash
pnpm build
```

Для разработки с автоматической пересборкой и встроенными source map используйте `pnpm dev`.

Production-команда создаёт минифицированный `main.js`. К GitHub Release нужно приложить `main.js`, `manifest.json` и `styles.css`; тег релиза должен точно совпадать с версией из `manifest.json`.
